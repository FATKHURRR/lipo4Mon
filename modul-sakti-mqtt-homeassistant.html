Integrasi Modul Sakti ke Home Assistant

Panduan lengkap mengubah data JSON yang dipublish Modul Sakti lewat MQTT menjadi entity sensor di Home Assistant — lengkap dengan struktur file, precision angka, dan troubleshooting.

Catatan: Contoh data pada tutorial ini menggunakan payload BMS Huawei sebagai ilustrasi. Struktur dan langkah yang sama berlaku untuk jenis data lain yang dipublish Modul Sakti — tinggal sesuaikan field JSON dan topic-nya.

Gambaran Umum

Modul Sakti mem-publish data ke MQTT broker dalam format JSON, satu topic per perangkat/sub-device yang terhubung ke logger. Contoh payload (data BMS Huawei, sebagai ilustrasi):

payload contoh — sysMon/9999999/AutoPoll/Huawei/0 
{
    "typeBattery": "Huawei",
    "AddrBattery": 214,
    "cell_count": 15,
    "cell_voltages": [3316, 3316, 3316, 3316, 3316, 3316, 3316, 3316, 3316, 3316, 3316, 3315, 3316, 3316, 3316],
    "cell_vmax": 3316,
    "cell_vmin": 3315,
    "cell_vdiff": 1,
    "bus_voltage": 49.81,
    "bat_voltage": 49.74,
    "bus_current": 0,
    "bat_current": -1.38,
    "full_capacity": 0,
    "rem_capacity": 0,
    "SOC": 90,
    "SOH": 97,
    "cycle_count": 957
}

Home Assistant tidak bisa otomatis "mem-parse" satu payload JSON jadi banyak sensor — tiap field harus didefinisikan manual sebagai satu mqtt sensor dengan value_template masing-masing.

Struktur Topic MQTT Modul Sakti

sysMon/9999999/AutoPoll/Huawei/0
        └ ID Modul Sakti (logger)
                     └ Jenis/merk perangkat
                             └ Address/index perangkat

Artinya:

  • 9999999 = ID Modul Sakti (satu unit bisa membaca banyak perangkat)
  • Huawei = jenis/merk perangkat yang terhubung (bisa berbeda tergantung perangkat yang dipasang)
  • 0 = address/index perangkat pada Modul Sakti tersebut

Karena satu Modul Sakti bisa membaca banyak perangkat, dan satu instalasi bisa punya banyak unit, konfigurasi dirancang satu file YAML per address perangkat — sehingga mudah diduplikasi saat menambah perangkat atau unit baru.

Step 1 — Setup Integrasi MQTT

Integrasi mqtt: di configuration.yaml hanya mendefinisikan entity, bukan koneksi ke broker. Koneksi broker diatur lewat UI:

  1. Settings → Devices & Services → Add Integration
  2. Cari MQTT
  3. Masukkan host broker, port, username, password (sesuaikan dengan broker yang dipakai Modul Sakti)
  4. Pastikan status Connected

Aktifkan auto-enable entity baru (penting!)

Secara default, entity baru dari integrasi manual bisa dibuat dalam status disabled. Untuk mencegah ini:

  1. Settings → Devices & Services
  2. Cari card MQTT → klik Configure
  3. Centang: Enable newly added entities dan Enable newly added devices
  4. Submit

Verifikasi data masuk

Developer Tools → MQTT → Listen to a topic, masukkan topic sesuai perangkat Modul Sakti kamu, contoh: sysMon/9999999/AutoPoll/Huawei/0. Klik Start Listening — kalau payload JSON muncul, berarti broker/topic sudah benar.

Step 2 — Konfigurasi configuration.yaml

configuration.yaml 
# Loads default set of integrations. Do not remove.
default_config:

frontend:
  themes: !include_dir_merge_named themes

automation: !include automations.yaml
script: !include scripts.yaml
scene: !include scenes.yaml

mqtt:
  sensor: !include_dir_merge_list mqtt/sensor/
Penting: gunakan !include_dir_merge_list, bukan !include_dir_list.
TagIsi tiap file yang diharapkan
!include_dir_list1 dict per file
!include_dir_merge_listlist of dict per file (semua digabung jadi 1 list besar)

Karena tiap file berisi banyak sensor (list - name: ...), tag yang benar adalah !include_dir_merge_list.

Step 3 — Struktur Folder

struktur direktori 
config/
├── configuration.yaml
└── mqtt/
    └── sensor/
        ├── sakti_9999999_huawei_0.yaml   ← Modul Sakti 9999999, perangkat Huawei address 0
        ├── sakti_9999999_huawei_1.yaml   ← Modul Sakti 9999999, perangkat Huawei address 1
        └── sakti_<id_modul_lain>_<jenis>_0.yaml  ← unit Modul Sakti lain

Path mqtt/sensor/ di configuration.yaml bersifat relatif terhadap folder config HA — pastikan foldernya persis di sebelah configuration.yaml.

Step 4 — File Sensor Lengkap (Contoh: BMS Huawei)

File berikut adalah contoh untuk perangkat BMS Huawei yang terbaca oleh Modul Sakti dengan ID 9999999. Mencakup semua field dari payload JSON, termasuk 15 cell voltage individual, dengan:

  • Voltage & current (pack/bus) → 2 desimal
  • SOC & SOH → 1 desimal
  • Cell voltage (individual, min/max/diff) → 3 desimal

Penjelasan pola konfigurasi

  • YAML anchor (&nama) dipakai di sensor pertama yang sudah lengkap (Voltage), lalu di-merge (<<: *nama) ke sensor berikutnya yang mirip — supaya tidak perlu tulis ulang field yang sama.
  • device: block memakai anchor terpisah supaya semua sensor dari satu perangkat otomatis dikelompokkan sebagai satu device di UI HA.
  • unique_id wajib unik di seluruh instalasi HA, formatnya: sakti_<id_modul>_<jenis_perangkat><address>_<metric>.
  • suggested_display_precision mengontrol jumlah desimal yang ditampilkan di UI — ini terpisah dari round() di value_template.
  • Field identitas statis (misal typeBattery, AddrBattery pada contoh ini) sengaja tidak dibuat jadi sensor karena sudah terwakili lewat nama device.

Step 5 — Validasi & Reload

Setiap kali mengubah file yaml:

  1. Developer Tools → YAML → Check Configuration (pastikan "Valid!")
  2. Developer Tools → YAML → MQTT → Reload (tidak perlu restart penuh)

Kalau ada error, baca pesannya — biasanya menyebutkan file dan baris yang bermasalah.

Step 6 — Enable Entity Otomatis

Kalau setting "Enable newly added entities" (Step 1) sudah dicentang sebelum reload pertama, seharusnya entity langsung aktif.

Kalau entity kadung dibuat dalam status disabled:

  1. Buka device terkait → klik link "+N disabled entities"
  2. Select all entity di daftar
  3. Klik Enable

Menambah Perangkat / Logger Baru

Perangkat baru di Modul Sakti yang sama (misal address 1)

  1. Copy file sakti_9999999_huawei_0.yaml → rename jadi sakti_9999999_huawei_1.yaml
  2. Find & replace di seluruh file: Huawei/0→Huawei/1, huawei0→huawei1, 9999999 Huawei 0→9999999 Huawei 1

Unit Modul Sakti baru (ID berbeda)

  1. Copy file sakti_9999999_huawei_0.yaml → rename jadi sakti_<ID_BARU>_huawei_0.yaml
  2. Find & replace 9999999 → ID Modul Sakti baru di seluruh file

Jenis perangkat berbeda

Kalau Modul Sakti membaca jenis perangkat lain (bukan BMS Huawei), sesuaikan:

  • Nama field di value_template sesuai struktur JSON perangkat tersebut
  • unit_of_measurement, device_class, dan icon sesuai jenis data
  • Bagian topic "Huawei" di path diganti sesuai identifier perangkat pada payload Modul Sakti

Jumlah cell berbeda (khusus contoh BMS)

Kalau baterai lain punya cell_count selain 15, tambah/kurangi blok Cell N Voltage sesuai kebutuhan (index array cell_voltages[N] dimulai dari 0).

Troubleshooting

Invalid config for 'mqtt': expected dict 'mqtt→0→sensor→0', got None

Penyebab: pakai !include_dir_list padahal isi tiap file adalah list sensor (pakai -), bukan satu dict.

Fix

Ganti jadi !include_dir_merge_list.

required key 'state_topic' not provided

Penyebab: ada item list yang jadi anchor "kosong" tanpa field lengkap. HA membaca semua item list sebagai sensor, termasuk anchor itu.

Fix

Anchor harus menempel pada sensor pertama yang sudah lengkap, bukan didefinisikan terpisah sebagai item kosong.

Sensor tampil "Unavailable"

Penyebab: MQTT sensor tidak fetch histori — hanya update state saat ada pesan baru masuk setelah entity dibuat/reload. Ini normal.

Fix

Tunggu sampai Modul Sakti publish data berikutnya. Kalau publish pakai retain: true, biasanya langsung terisi saat reload.

Angka tidak muncul desimalnya walau sudah pakai round()

Penyebab: round() di value_template hanya memengaruhi nilai state yang disimpan, tapi tampilan di UI dikontrol terpisah oleh Display Precision di entity registry — yang di-cache saat entity pertama kali dibuat.

Fix
  1. Tambahkan suggested_display_precision: N di config sensor
  2. Hapus entity/device yang sudah terlanjur dibuat (Settings → Devices & Services → Devices → cari device → menu titik tiga → Delete)
  3. Reload MQTT, tunggu publish berikutnya — entity ter-generate ulang dengan precision baru
Entity baru berstatus disabled

Penyebab: setting "Enable newly added entities" belum dicentang saat entity pertama kali dibuat.

Fix

Cek Settings → Devices & Services → MQTT → Configure → centang "Enable newly added entities" dan "Enable newly added devices". Untuk entity yang sudah kadung disabled, enable manual lewat link "+N disabled entities" di halaman device.
